# ADR-0003 — TBPs and Issue Templates Are Not Defaults

## Status

Accepted.

## Context

TBPs and issue templates became process-heavy indirection. They made implementation agents read methodology instead of localized project truth.

## Decision

TBPs and issue templates are not default repository layers.

They may be activated only by a profile or project-type guide when a repository has a proven process-heavy need.

## Consequences

- Milestones and specs carry implementation work.
- Documentation sync is a separate task type.
- Repositories become smaller and easier for agents to navigate.
